帧的接收：net_rx_action

NIC 接收帧基本就是最常用的就是两种方式：中断和轮询。中断的方式就是 NIC 接收到帧之后，通过 DMA 的方式
将帧放到 poll_list 中，然后发起一个中断信号引起 CPU 注意触发中断处理流程。轮询就是隔一段时间自动查询
poll_list ，间隔时间大概在 100ms，两种方式各有优缺点。

接收到中断事件的 CPU 会调用 do_IRQ 函数。IRQ 变化会引发正确的处理函数被启用。此处理函数通常是设备
驱动程序初始化期间所注册的函数。IRQ 函数处理例程会在中断的模式下执行。
